## Grafana & Prometheus

These two tools are almost always used together as a monitoring and observability stack, but they do different jobs.

---

### Prometheus

Prometheus is a **time-series database and metrics collection system**. Its job is to gather, store, and query numeric metrics over time.

**How it works:**
- It uses a **pull model** — Prometheus actively scrapes metrics from your services at regular intervals (e.g. every 15s), rather than services pushing data to it
- Your apps expose a `/metrics` HTTP endpoint (either natively or via an **exporter**), and Prometheus pulls from it
- Data is stored as time-series: a metric name + labels + a value + a timestamp
- You query it using **PromQL**, its own query language

**What it monitors:** CPU, memory, request rates, error rates, latency, queue depths — basically any numeric measurement you care about.

**Exporters** are small adapters that translate third-party systems into Prometheus format — there are exporters for Postgres, Redis, Nginx, Kubernetes nodes, etc.

---

### Grafana

Grafana is a **visualization and dashboarding platform**. It doesn't store data itself — it connects to data sources and renders them.

**How it works:**
- You point Grafana at one or more data sources (Prometheus being the most common)
- You build dashboards with panels: graphs, gauges, tables, heatmaps, etc.
- Each panel runs a query (PromQL for Prometheus) and visualizes the result
- You can set up **alerts** that fire when a metric crosses a threshold, notifying you via Slack, PagerDuty, email, etc.

---

### How they fit together

```
Your App / Infrastructure
        │
        │  (Prometheus scrapes /metrics every N seconds)
        ▼
   Prometheus  ◄──── stores time-series data, evaluates alert rules
        │
        │  (Grafana queries via PromQL)
        ▼
    Grafana  ──────► Dashboards, alerts, visualizations
```

A typical setup:
1. Your service exposes metrics (or you deploy an exporter)
2. Prometheus scrapes and stores them
3. Grafana visualizes them in a dashboard
4. Grafana (or Prometheus Alertmanager) fires alerts when something looks wrong

---

### Why this combo is popular

- Both are **open source** and have huge ecosystems
- Prometheus is the de facto standard in **Kubernetes/cloud-native** environments
- Grafana supports **many other data sources** too (Loki for logs, Tempo for traces, InfluxDB, CloudWatch, etc.), so it naturally becomes your single observability UI
- Together they cover the **metrics** pillar of observability well

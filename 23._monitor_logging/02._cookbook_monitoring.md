# Monitoring in Your Own Project

## Assignment

**Type:** Group work  
**Deadline:** Soft deadline — should be done before next session  
**Part of the exam report:** Yes  
**Requirement:** Your monitoring setup must be running and accessible

---

## Goal

Add a monitoring stack to the `awsome_recipe_cookbook` project so that you have visibility into the health and behavior of your running application.

---

## Tasks

### 1. Add a `/metrics` endpoint to your application

Instrument your application with a Prometheus client library appropriate for your language/framework (see the [tutorial](01._prometheus_grafana_setup.md) for examples).

At minimum expose:
- Default/runtime metrics (memory, CPU, GC if applicable)
- A counter for total HTTP requests, labelled by route and status code

### 2. Add Prometheus and Grafana via Docker Compose

Add a `monitoring/` directory to your repository containing:
- `prometheus.yml` configured to scrape your application
- A `docker-compose.yml` (or extend your existing one) that starts Prometheus and Grafana alongside your app

Your full stack should be startable with a single `docker compose up` command.

### 3. Create a Grafana dashboard

Build a dashboard in Grafana that shows at least:
- Whether your application is up (`up` metric)
- Request rate over time
- One metric that is meaningful and specific to your application (e.g. recipes created, search queries, errors)

Export the dashboard as JSON and commit it to your repository so it can be re-imported.

### 4. Document it

Add a short section to your project `README.md` explaining:
- How to start the monitoring stack
- Where to access Grafana and what the dashboard shows

---

## Hints

* If your app and monitoring stack are in separate Compose files, use `docker compose -f docker-compose.yml -f monitoring/docker-compose.yml up`.
* On Linux, `host.docker.internal` may not resolve — use your host's LAN IP or put the app itself in the Compose network.
* Grafana dashboards can be provisioned automatically on startup — look into [Grafana provisioning](https://grafana.com/docs/grafana/latest/administration/provisioning/) if you want to avoid manual re-import.

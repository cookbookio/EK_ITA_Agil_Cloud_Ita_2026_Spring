# Monitoring a Multi-VM Deployment on Azure

This tutorial extends [01._prometheus_grafana_setup.md](01._prometheus_grafana_setup.md) to a two-VM setup on Azure: an nginx reverse proxy and a backend.

## Architecture

```
[nginx VM]       node_exporter :9100
                 nginx-prometheus-exporter :9113
                 Prometheus :9090        ← runs here too
                 Grafana    :3000

[backend VM]     node_exporter :9100
                 app /metrics :8080  (see tutorial 01)
```

> **Where to run Prometheus/Grafana?**  
> A dedicated monitoring VM is cleanest, but costs money. Running on the nginx VM is practical: it is already your network entry point and always running.

---

## 1. Open Azure NSG Ports

Each VM needs to allow inbound traffic on the exporter ports **from the nginx VM's private IP only** (not `*`).

| Port | Purpose | Open on |
|------|---------|---------|
| 9100 | node_exporter | both VMs |
| 9113 | nginx-prometheus-exporter | nginx VM |
| 8080 | backend /metrics | backend VM |
| 9090 | Prometheus UI | nginx VM (your IP only) |
| 3000 | Grafana UI | nginx VM (your IP only) |

In the Azure portal: **VM → Networking → Add inbound port rule**.  
For scraping ports (9100, 8080), set Source to the **private IP** of your nginx VM.

---

## 2. Install node_exporter on Both VMs

Run this on **both the nginx VM and the backend VM**:

```bash
wget https://github.com/prometheus/node_exporter/releases/latest/download/node_exporter-1.9.1.linux-amd64.tar.gz
tar xvf node_exporter-*.tar.gz
sudo cp node_exporter-*/node_exporter /usr/local/bin/

sudo tee /etc/systemd/system/node_exporter.service <<EOF
[Unit]
Description=Node Exporter
After=network.target

[Service]
ExecStart=/usr/local/bin/node_exporter

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now node_exporter
```

Verify it works:
```bash
curl http://localhost:9100/metrics | head -5
```

---

## 3. Expose Nginx Metrics

Nginx does not expose metrics by default. Enable `stub_status` and run `nginx-prometheus-exporter` to translate it for Prometheus.

**Step 1 — Enable stub_status in nginx** (on the nginx VM):

```nginx
# Add this to /etc/nginx/conf.d/stub_status.conf
server {
    listen 127.0.0.1:8081;
    location /stub_status {
        stub_status;
        allow 127.0.0.1;
        deny all;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
```

**Step 2 — Install nginx-prometheus-exporter** (on the nginx VM):

```bash
wget https://github.com/nginx/nginx-prometheus-exporter/releases/latest/download/nginx-prometheus-exporter_1.4.2_linux_amd64.tar.gz
tar xvf nginx-prometheus-exporter_*.tar.gz
sudo cp nginx-prometheus-exporter /usr/local/bin/

sudo tee /etc/systemd/system/nginx_exporter.service <<EOF
[Unit]
Description=Nginx Prometheus Exporter
After=network.target

[Service]
ExecStart=/usr/local/bin/nginx-prometheus-exporter \
  --nginx.scrape-uri=http://127.0.0.1:8081/stub_status

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now nginx_exporter
```

Verify:
```bash
curl http://localhost:9113/metrics | grep nginx
```

## 4. Configure Prometheus on the Nginx VM

Create `monitoring/prometheus.yml` with the **private IPs** of each VM:

```yaml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: "node"
    static_configs:
      - targets:
          - "10.0.0.4:9100"   # nginx VM
          - "10.0.0.5:9100"   # backend VM

  - job_name: "nginx"
    static_configs:
      - targets: ["10.0.0.4:9113"]

  - job_name: "backend"
    static_configs:
      - targets: ["10.0.0.5:8080"]
    metrics_path: "/metrics"
```

Replace the IPs with your actual Azure private IPs (found in the Azure portal under each VM's **Networking** tab).

## 5. Run Prometheus and Grafana

Create `monitoring/docker-compose.yml` on the nginx VM (same as [tutorial 01](01._prometheus_grafana_setup.md)):

```yaml
services:
  prometheus:
    image: prom/prometheus:latest
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    ports:
      - "9090:9090"

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
    depends_on:
      - prometheus
```

```bash
docker compose up -d
```

Check that all targets are healthy at **http://\<nginx-public-ip\>:9090/targets** — every target should show `UP`.

## Troubleshooting

**Target shows `DOWN` in Prometheus:**
- SSH into the nginx VM and `curl http://<target-private-ip>:<port>/metrics` — if it times out, the NSG rule is missing or wrong.
- If it refuses connection, the exporter service is not running: `sudo systemctl status node_exporter`.

**`host.docker.internal` does not resolve on Linux:**
- Use the VM's private IP directly, or add `extra_hosts: ["host.docker.internal:host-gateway"]` to the Prometheus service in docker-compose.yml.

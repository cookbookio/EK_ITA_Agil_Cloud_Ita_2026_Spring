# Setting up Prometheus and Grafana

This tutorial walks you through getting a basic monitoring stack running locally using Docker Compose.

## Prerequisites

* Docker and Docker Compose installed
* A running application that exposes an HTTP endpoint

---

## 1. Project Structure

Create a `monitoring/` directory in your project with the following structure:

```
monitoring/
├── docker-compose.yml
└── prometheus.yml
```

---

## 2. Prometheus Configuration

Create `monitoring/prometheus.yml`:

```yaml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: "your-app"
    static_configs:
      - targets: ["host.docker.internal:8080"]
```

Replace `8080` with the port your application runs on.

> `host.docker.internal` lets Prometheus inside Docker reach your app running on the host machine. On Linux you may need to use your machine's local IP instead.

---

## 3. Docker Compose

Create `monitoring/docker-compose.yml`:

```yaml
services:
  prometheus:
    image: prom/prometheus:latest
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    ports:
      - "9090:9090"

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
    depends_on:
      - prometheus
```

Start the stack:

```bash
docker compose up -d
```

* Prometheus UI: http://localhost:9090
* Grafana UI: http://localhost:3000 (login: `admin` / `admin`)

---

## 4. Exposing Metrics from Your Application

Prometheus scrapes a `/metrics` endpoint in its own text format. Most frameworks have a library for this:

| Language | Library |
|----------|---------|
| Python   | `prometheus-client` |
| Node.js  | `prom-client` |
| Go       | `prometheus/client_golang` |
| Java     | `micrometer-registry-prometheus` |

**Example — Python:**

Install the library first:

```bash
pip install prometheus-client
```

The example below is a **standalone demo script** (`metrics.py`). Run it on its own (`python metrics.py`) to verify that Prometheus can scrape your machine before touching your real app. It is not meant to run alongside your app — it *is* the app in this case.

```python
import time
from prometheus_client import Counter, start_http_server

REQUEST_COUNT = Counter("http_requests_total", "Total HTTP requests")

start_http_server(8080)  # exposes /metrics on port 8080

print("Metrics server running on http://localhost:8080/metrics")

while True:
    REQUEST_COUNT.inc()
    time.sleep(5)
```

> The `while True` loop keeps the process alive so the metrics server stays up. `REQUEST_COUNT.inc()` simulates activity so you can see the counter climb in Grafana.


**Example — Node.js (inside your existing `app.js` or `server.js`):**

This snippet adds a `/metrics` route to your existing Express app. You do not create a separate file — just add the lines to your main server file. Install the library first:

```bash
npm install prom-client
```

```js
const client = require("prom-client");
const register = new client.Registry();
client.collectDefaultMetrics({ register });

app.get("/metrics", async (req, res) => {
  res.set("Content-Type", register.contentType);
  res.end(await register.metrics());
});
```

---

### Final file structure

After completing all steps your project should look something like this:

```
your-project/
├── app.py / app.js          ← your application (metrics endpoint added here)
├── requirements.txt / package.json
└── monitoring/
    ├── docker-compose.yml   ← runs Prometheus + Grafana
    └── prometheus.yml       ← tells Prometheus where to scrape
```

Start your app first, then run `docker compose up -d` from the `monitoring/` folder. Prometheus will scrape your app's `/metrics` endpoint, and Grafana will visualize the data.

## 5. Connecting Grafana to Prometheus

1. Open Grafana at http://localhost:3000 and log in (`admin` / `admin`)
2. In the left sidebar go to **Connections → Data sources**
3. Click **Add data source**
4. From the list of data source types, select **Prometheus** (under *Time series databases*)
5. In the **Connection** section set the Prometheus server URL to `http://prometheus:9090`
6. Scroll down and click **Save & test** — you should see a green confirmation message

> If you get a "Please enter a valid URL" error, make sure the URL is `http://prometheus:9090` (not `localhost`) so Grafana can reach Prometheus inside the Docker network.

---

## 6. Creating a Dashboard

1. In the left sidebar go to **Dashboards**
2. Click **+ Create dashboard** (or **New → New dashboard** from the top-right button)
3. A new empty dashboard opens — click the **+ (Add)** button and choose **Panel** to add a visualization
4. In the query editor at the bottom, select your Prometheus data source and enter a metric such as `up` or `http_requests_total`
5. Click **Apply**, then **Save** (top-right) to save the dashboard

You can also import pre-built dashboards from [grafana.com/grafana/dashboards](https://grafana.com/grafana/dashboards) using their dashboard ID.
